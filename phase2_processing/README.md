# Phase 2 — Processing & Storage Layer
## AI-Powered Review Discovery Engine | Spotify Growth Team

---

## Overview

This directory contains the complete **Phase 2 implementation** of the AI-Powered Review Discovery Engine. This layer acts as a data processor and indexer that cleans, enriches, and indexes the raw review inputs generated by Phase 1.

It runs as a daemon process (`pipeline_runner.py`) that constantly or periodically polls the database for unprocessed reviews, passes them through a 7-step pipeline, and prepares them for semantic retrieval in the RAG layer (Phase 3).

---

## 7-Step Cleaning & Indexing Pipeline

```
 PostgreSQL (pending)
         │
         ▼
 1. Deduplication (MD5 hashing)
         │
         ▼
 2. Language Detection (langdetect)
         │
         ▼
 3. Noise Filter (exclude short/spam/tech issues)
         │
         ▼
 4. Relevance Scorer (music discovery relevance [0.0 - 1.0])
         │
         ▼
 5. Enrichment (add word_count, review_age, platform_weight)
         │
         ▼
 6. Embedding Generator (vector embeddings)
         │
         ▼
 7. Vector DB Indexing (ChromaDB / Pinecone / PostgreSQL fallback)
         │
         ▼
 PostgreSQL (processed)
```

---

## Zero-Dependency Fallbacks (Out-of-the-Box Local Development)

AI pipelines are often complex to configure locally due to external API keys or heavy local C++ dependencies (e.g. compiling PyTorch for embeddings or ChromaDB). 

To ensure this pipeline runs instantly on any local developer setup without requiring API keys or heavy packages, the engine is designed with **robust defensive fallbacks**:

| Pipeline Stage | Primary Implementation | Fallback System (Automatic) |
|---|---|---|
| **Relevance Scorer** | Google Gemini 2.5 API | Weighted keyword density and frequency analysis (returns heuristic score between 0.0 - 1.0). |
| **Embedding Gen** | Local `sentence-transformers` or OpenAI/Gemini Embeddings API | Deterministic, hash-based dummy float vector (384 dimensions) generated from text salt. |
| **Vector DB Store** | ChromaDB Container or Pinecone Cloud | Dedicated PostgreSQL table (`raw_reviews_embeddings`) which uses JSONB to index vectors. |

## Quick Start (Local Run)

### Prerequisites
- Python 3.10+
- The Phase 1 database must be running (e.g. via `docker-compose up -d` in `phase1_ingestion`).

### Step 1 — Setup Environment
Make sure your environment variables point to your local PostgreSQL database (matches the values in Phase 1's `.env`):
```bash
# Windows (PowerShell)
$env:DATABASE_URL="postgresql://review_engine:ReviewEngine2026!@localhost:5432/reviews_db"

# Optional: Add your Google Gemini API Key to enable LLM-based Relevance Scoring & Embeddings
$env:GEMINI_API_KEY="AIzaSyB_..."
```

### Step 2 — Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 3 — Run the Pipeline
Run the daemon process. It will automatically wait for incoming database records from Phase 1, clean them, generate vectors, and mark them as `processed`:
```bash
python pipeline_runner.py
```

## Database Status Codes

After running, check the status of your reviews in PostgreSQL:
```sql
SELECT processing_status, COUNT(*) FROM raw_reviews GROUP BY processing_status;
```

*   `pending`: Ready for Phase 2 processing.
*   `processing`: Locked by the processing loop.
*   `processed`: Successfully cleaned, enriched, and indexed in the vector store.
*   `excluded`: Filtered out due to duplicate content, non-English, spam, or low relevance.
*   `failed`: An unexpected error occurred; check the `error_message` column for details.
